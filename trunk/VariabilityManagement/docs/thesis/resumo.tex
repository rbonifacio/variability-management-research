\resumo
Com o objetivo de reduzir o \emph{time-to-marketing} e melhorar a qualidade dos
produtos de software, diferentes técnicas para economia de escopo, customização
em massa e reuso sistmático foram propostas. Exemplos dessas abordagens incluem
Linhas de Produtos de Software, as Técnicas de Programação Generativa, e as
Fábricas de Software. Existem pontos em comum entre essas abordagens, como a
análise de domínio que visa definir o escopo (frequentemente no sentido
negocial) no qual artefatos reusáveis pode ser usados para gerar produtos
específicos. Adicionalmente, o uso de modelo de features é uma prática comum
para representar features comuns at dosos os produtos pertencentes aos escopo e
quais features são opcionais, sendo útil para diferenciar produtos específicos
de uma mesma famíla. Consequentemente, para gera produtos específicos, faz-se
necessário: (a) introduzir os pontos de variação adequatos nos artefatos
comuns, (b) desenvolver artefatos que extendem os pontos de variação e (c)
relacionar as features aos artefatos comuns e variáveis. Nesta tese,
consideramos a disciplina de \emph{Gerência de Variação} como sendo responsável
por essas atividades. Tem sido observado que a gerência de variação corresponde
a um tipo interessante de preocupação transversal, uma vez que algumas features
requerem o espalhamento de pontos de variação em diferentes artefatos
de requisitos, design, código e testes. A natureza transversal da gerência de
variação resulta em desafios interessantes relacionados com rastreabilidade,
evolução e geração de produtos específicos. Por esse motivo, tem sido
investigado o uso de técnicas orientadas a aspectos para melhorar a
modularidade dos mecanismos de composição entre os artefatos comuns e variantes
de uma linha de produto. Nesta tese, propomos uma visão mais abrangente da
gerência de variações, descrevendo sua semântica como mecanismos crosscutting
ao mesmo tempo que detalhamos a contribuição de diferentes artefatos no
processo de geração de produtos específicos.
\begin{keywords}
Linhas de produto de software, Gerência de variação, Mecanismos de composição
transversais.
\end{keywords}
